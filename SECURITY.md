# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest | ✅ Security patches |
| Older | ❌ Upgrade recommended |

## Reporting a Vulnerability

Report security vulnerabilities via email to **arjunaajalahla100@gmail.com**.

Please do **not** report security vulnerabilities through public GitHub issues.

### What to Include

- Type of issue
- Affected source files (paths or URLs)
- Steps to reproduce
- Proof-of-concept if available
- Impact assessment

### Response Timeline

1. Acknowledgment within 24 hours
2. Triage within 72 hours
3. Fix typically within 7-14 days for critical issues
4. Coordinated public disclosure after release

## Responsible Disclosure

We request a 90-day embargo period from the initial report to allow time for a fix before public disclosure.
